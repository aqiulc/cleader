# cleader v0.4.4 — Grid Library View with Cover Cache and Lazy Generation

**Date:** 2026-05-11
**Branch:** `impl/v0.1.0`
**Predecessor:** v0.4.3 (cover ASCII art for single books) + v0.4.3.5 (return to library on reader quit)

## Goal

When the user opens a directory as a library, render a grid of book covers — one cell per book, ASCII-art thumbnails generated on demand for visible cells only. Covers persist to disk so re-opens are instant. A plain list view remains available, with the chosen view mode remembered across sessions.

## Why

- **Visual break in a text-only interface.** Covers add personality to a directory listing without changing the keyboard-first workflow.
- **Discoverability.** A user with hundreds of books recognizes covers faster than they parse titles.
- **Scalability.** Viewport-driven generation means a 10,000-book directory uses the same CPU as a 16-book directory: only what's on screen is decoded.

## Non-goals

- **No inline-image ASCII art in chapters.** Still tracked as a separate v0.4.x item.
- **No real logo.** A placeholder ASCII pattern ships with the code; the constant is swappable when a logo exists.
- **No cache eviction / size cap.** Covers are ~400 bytes each; 2500 books ≈ 1 MB. Revisit only if a real user hits a size problem.
- **No prefetch.** Worker generates strictly what's visible. Off-screen books wait.
- **No threading beyond one worker.** Single worker thread per library session is enough at ~30 ms per cover.

## Architecture

### New modules

#### `src/cover_cache.rs`

Owns three things:

1. **Memory map:** `HashMap<BookId, CoverState>` where `CoverState = Pending | Ready(Vec<String>)`.
2. **Disk store:** `<data_dir>/covers/<book_id>.txt` (one cover per file, plain text, one line per row).
3. **Background worker:** A thread that consumes `(BookId, PathBuf)` jobs from an `mpsc::Receiver`, decodes the EPUB cover, calls `image_to_ascii(bytes, COVER_THUMBNAIL_WIDTH)`, writes to disk, and sends the result back on a result channel.

Public API:

```rust
pub struct CoverCache { /* private */ }

impl CoverCache {
    /// Spawn worker thread and channels. Cheap; library_app should call once per session.
    pub fn new() -> Self;

    /// Pull any finished covers from the worker. Call once per frame before reading from the cache.
    /// Returns true if any new covers landed (used to trigger a redraw).
    pub fn drain_finished(&mut self) -> bool;

    /// Get a cover for rendering. Returns `Some(&[String])` only if Ready in memory.
    /// Pending or Miss returns `None` (caller renders placeholder).
    pub fn get(&self, book_id: &BookId) -> Option<&[String]>;

    /// Request a cover. No-op if already Ready or Pending. On a miss, checks disk
    /// synchronously (fast read); on disk miss, enqueues the worker. Idempotent.
    pub fn enqueue(&mut self, book_id: BookId, epub_path: PathBuf);
}

impl Drop for CoverCache {
    /// Drop the job sender so the worker thread exits cleanly.
    fn drop(&mut self);
}
```

Internal job/result types:

```rust
struct CoverJob { book_id: BookId, epub_path: PathBuf }
struct CoverResult { book_id: BookId, lines: Vec<String> }
```

Disk layout: `<data_dir>/covers/<hex_book_id>.txt`. One row per line, exactly `COVER_THUMBNAIL_WIDTH` chars wide. Errors during write are non-fatal (log and continue — memory cache still serves the session).

Constants in this module:
- `COVER_THUMBNAIL_WIDTH: u16 = 22` (cell content area; cell itself is 24 including 1-col border on each side)
- `COVER_THUMBNAIL_HEIGHT_HINT: u16 = 12` (rows; final height derived from source aspect by `image_to_ascii`, clamped to ≤ 12 for layout consistency — if a cover renders fewer rows, pad with blank lines; if it would render more, truncate to 12)
- `PLACEHOLDER: &[&str]` — 12-row × 22-col ASCII pattern shown for Pending and Miss states. Centered "cleader" wordmark inside a framed box; same dimensions as a real cover so cell layout never shifts when a cover arrives.

#### `src/prefs.rs`

Small JSON-on-disk for user preferences. Mirrors `persistence.rs` shape (load_from / save_to / open / open_at), but simpler — no migration concerns yet.

```rust
#[derive(Serialize, Deserialize, Default, Clone)]
pub struct Prefs {
    pub view_mode: ViewMode,  // defaults to ViewMode::Grid
}

#[derive(Serialize, Deserialize, Clone, Copy, PartialEq, Eq)]
#[serde(rename_all = "lowercase")]
pub enum ViewMode { Grid, List }

impl Default for ViewMode { fn default() -> Self { ViewMode::Grid } }

pub struct PrefsStore { /* path + cached prefs */ }

impl PrefsStore {
    pub fn open() -> Result<Self, PrefsError>;
    #[doc(hidden)]
    pub fn open_at(path: PathBuf) -> Self;
    pub fn view_mode(&self) -> ViewMode;
    pub fn set_view_mode(&mut self, mode: ViewMode) -> Result<(), PrefsError>;  // writes immediately
}
```

Path: `<data_dir>/prefs.json` next to `registry.json`. Default-on-missing-file (mirrors `Persistence` recovery semantics).

### Changes to existing modules

#### `src/library_app.rs`

Add fields:
- `view_mode: ViewMode`
- `cover_cache: CoverCache`
- `prefs: PrefsStore`
- `save_error: Option<String>` (for prefs flush failures — surface in status bar like `App::save_error`)

New methods:
- `cover_cache(&self) -> &CoverCache` (read access for renderer)
- `cover_cache_mut(&mut self) -> &mut CoverCache` (for `drain_finished` from main loop)
- `view_mode(&self) -> ViewMode`
- `toggle_view_mode(&mut self)` — flips field, writes prefs, captures error to `save_error`
- `request_visible_covers(&mut self, visible: impl IntoIterator<Item = usize>)` — for each entry index, `cover_cache.enqueue(...)`

Action handling:
- New `Action::ToggleViewMode` arm calls `toggle_view_mode()`
- Existing scroll math (PageUp/PageDown) needs to be aware of view mode: in grid mode, page = visible rows × cols; in list mode, page = visible rows (unchanged)

#### `src/input.rs`

Add `Action::ToggleViewMode` to the enum (already `#[non_exhaustive]`, so this is non-breaking).

Bind `g` to `Action::ToggleViewMode` in `translate()`. This is library-only — reader's `App::handle` should ignore the variant (existing match arms cover only what they care about; an unmatched arm falls through by default).

#### `src/reader.rs`

Refactor `render_library` into two functions:
- `render_library_list` — current behavior, unchanged
- `render_library_grid` — new: takes the same `LibraryRenderInput` plus a `&CoverCache`. Computes columns from `area.width / CELL_WIDTH`, lays out cells in a `Layout::vertical` of cell-rows, each row split into `Layout::horizontal` cells. Each cell is a `Block::default().borders(Borders::ALL)` with the cover lines as a `Paragraph` in the upper region and title/author as a separate inner block in the lower region.

Dispatcher: `render_library` becomes a thin function that picks list or grid based on `LibraryRenderInput.view_mode`. Add `view_mode: ViewMode` and `cover_cache: Option<&CoverCache>` to `LibraryRenderInput`. (List view doesn't need the cache; passing `None` is fine.)

Cell layout (24 cols × 16 rows):
- Outer `Borders::ALL` → 1-col padding on left/right/top/bottom
- Inner area: 22 cols × 14 rows
- Top sub-block (cover): 22×12 — `Paragraph::new(cover_lines)`
- Bottom sub-block (title/author): 22×2 — `Borders::TOP` (a single horizontal divider, no left/right re-borders), two `Line`s (title — bold or reverse if selected; author — DIM)

Visible-entry computation:
- `visible_first = scroll_offset` (row index of top row × cols_per_row)
- `visible_last = visible_first + (rows_per_screen × cols_per_row) - 1, clamped to entries.len() - 1`
- Pass `visible_first..=visible_last` to `library_app.request_visible_covers`

Selected cell:
- Border style swaps to `Color::Yellow` (matches TOC selection convention)
- Title line gets `Modifier::REVERSED`

Tiny terminals (width < `CELL_WIDTH = 24`, or height < `CELL_HEIGHT = 16`): fall back to list view rendering, ignoring the view_mode field. Cells are not splittable below those dimensions without clipping content. Lock this in with a `TestBackend` test at 10×3 and 24×3.

#### `src/main.rs`

Modify `library_event_loop`:

```rust
fn library_event_loop(...) -> anyhow::Result<()> {
    while !app.should_quit() {
        // Drain any finished covers from worker thread, mark dirty if needed.
        let cover_changed = app.cover_cache_mut().drain_finished();

        terminal.draw(|frame| {
            // ... existing render call, now with view_mode + cover_cache in input ...
            //
            // Identify visible entries from the rendered layout (or recompute the math
            // here) and call app.request_visible_covers(visible_range).
        })?;

        // Poll with 50 ms timeout so the worker has a chance to deliver covers
        // even when no user input arrives.
        if event::poll(std::time::Duration::from_millis(50))? {
            let evt = event::read()?;
            if let Some(action) = translate(evt) {
                app.handle(action);
            }
        }
        // else: timeout, loop back and drain again
    }
    Ok(())
}
```

The outer `run_library_session` loop is unchanged — the cache lives on the LibraryApp, which itself outlives reader sessions, so covers persist in memory across "library → reader → back to library" cycles.

#### `src/lib.rs`

Add `pub mod cover_cache;` and `pub mod prefs;`.

## Data flow

1. **Library open:**
   - `scan_directory` returns entries (existing).
   - `PrefsStore::open()` loads `view_mode` (defaults to Grid).
   - `LibraryApp::new(entries, viewport)` creates a `CoverCache` (spawning the worker thread) and stores the prefs.

2. **First draw:**
   - `drain_finished()` — nothing yet, returns false.
   - Render the grid with placeholders for all visible cells.
   - `request_visible_covers(visible)` — for each visible BookId, `cache.enqueue(...)`. First call hits disk (likely miss on first-ever open), then queues the worker.

3. **Worker tick:** Receives a job, decodes the EPUB cover, calls `image_to_ascii`, writes to disk, sends back the result.

4. **Subsequent draws:**
   - `drain_finished()` — pulls any finished covers into the mem map, returns true.
   - Redraw shows real cover for those cells, placeholders for the rest.

5. **User scrolls:**
   - `handle(Action::ScrollDown)` updates `selection` and `scroll_offset` (existing logic, with cols-aware step in grid mode).
   - Next draw computes a new visible range. New uncached BookIds get enqueued. In-flight work for the old viewport finishes naturally and lands in the mem map for later.

6. **User presses `g`:**
   - `handle(Action::ToggleViewMode)` flips `view_mode`, calls `prefs.set_view_mode(...)`, captures any write error in `save_error`.
   - Next draw switches dispatchers. In list mode, the cache idles (no enqueue calls) but isn't dropped — toggling back is instant for already-Ready covers.

7. **User picks a book:**
   - Existing flow: `selected_path` set, library_event_loop returns, reader runs.
   - Cache stays on `LibraryApp` (which outlives the reader session).

8. **Reader exits:**
   - Library resumes with cache and prefs intact. New visible entries (if terminal was resized) get enqueued.

9. **User quits the library entirely:**
   - `LibraryApp` drops → `CoverCache` drops → job channel closes → worker thread sees `RecvError` and exits.

## Error handling

- **Cover decode failure** (e.g., EPUB has no cover, or cover is a format `image` can't handle): worker sends back `Vec<String>` of placeholder lines (so cell renders placeholder instead of looking broken). Decode errors are non-fatal; we proceed without populating disk for that BookId.
- **Disk write failure** (e.g., disk full): worker logs (via `eprintln!` on a separate stderr channel? no — log to a debug buffer if we add one. For v0.4.4: silent. The next session will re-decode.). The result still goes to mem, so this session is unaffected.
- **Prefs write failure:** captured in `LibraryApp.save_error` and shown in the library status bar, identical to how `App.save_error` works in the reader.
- **Disk read failure** during `enqueue` (corrupt cache file): treat as miss; re-enqueue the worker to regenerate.

## What ships at the end of v0.4.4

- `cleader path/to/dir/` opens directly into grid view (or list, if user previously toggled).
- Pressing `g` flips between grid and list, persisted across sessions.
- First time a directory is opened: grid renders immediately with placeholders, covers fill in as the worker generates them (~30 ms each, viewport-only).
- Second time: covers load from disk almost instantly.
- Move the EPUB file: cover stays cached (BookId is content-hashed).
- Scroll a huge library: only visible covers generate; off-screen books never compute.
- Reader exit returns to library with cache intact.
